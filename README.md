# 🚀 任務儀表板與問題追蹤工具 (Vibe-Pro Task Tracker)

這是一個基於 **React** 與 **Tailwind CSS** 開發的極簡風格待辦事項工具，特別針對「開發紀錄」與「問題解決」進行了優化。透過 **Google Apps Script** 實現了免伺服器的雲端資料同步。

## 📌 專案願景
不僅僅是管理「要做什麼」，更專注於記錄「遇到了什麼困難」以及「最後如何解決」。這是一個結合了 **待辦清單 (To-do List)** 與 **輕量級問題追蹤 (Issue Tracker)** 的整合方案。

---

## ✨ 核心功能
- **視覺化任務管理**：採用靛藍色系與薄荷蘇打配色，支援多級優先度（🔥 緊急、⚡ 普通、🌱 低優先、🏠 家務）。
- **問題追蹤系統**：每個任務可獨立紀錄多筆 Q&A，方便回溯開發痛點。
- **進度即時監控**：滑桿式進度調整，配合動態進度條顯示。
- **雲端雙向同步**：自動同步至 Google Sheets，換台電腦、打開手機也能無縫銜接。
- **響應式設計**：支援桌面側欄模式與行動裝置緊湊視圖。

## 📈 開發歷程摘要

### 1. 基礎架構 (v1.0)
- 建立 React 核心邏輯，實作基本的 CRUD (增刪改查) 功能。
- 配置 GAS 腳本，解決跨網域 (CORS) 寫入 Google 試算表的問題。

### 2. UI/UX 深度優化 (v2.0)
- 導入 **Glassmorphism (玻璃擬態)** 彈窗設計。
- 為了增加 scannability (可讀性)，將字體與卡片比例進行緊湊化調整。

### 3. 問題紀錄模組 (v2.0.5)
- 新增任務詳情內的 Issue Tracker，支援 JSON 格式儲存複雜的問答對。
- 解決了 JSON 序列化在試算表欄位中的對位問題。

### 4. 穩定性修復與部署 (v2.0.7)
- **事件冒泡修復**：使用 `e.stopPropagation()` 解決刪除按鈕與卡片點擊事件的衝突。
- **環境適配**：確認並排除 VS Code Live Preview 的快取干擾，確保在生產環境 (GitHub Pages) 穩定運作。

## 💡 使用說明
1. **新增任務**：點擊右上角「+ 新增任務」，設定名稱與優先級。
2. **追蹤問題**：點擊任務卡片進入詳情，點擊「+ 新紀錄」寫下遇到的問題。
3. **完成任務**：點擊卡片右上角勾選框，或在詳情中點擊「標記為完成」。
4. **刪除**：進入詳情頁面，點擊右下角「刪除任務」並確認。
