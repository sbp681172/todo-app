# 🚀 任務儀表板與問題追蹤工具 (Vibe-Pro Task Tracker)

這是一個基於 **React** 與 **Tailwind CSS** 開發的極簡風格待辦事項工具，特別針對「開發紀錄」與「問題解決」進行了優化。透過 **Google Apps Script** 實現了免伺服器的雲端資料同步。

## 📌 專案願景
不僅僅是管理「要做什麼」，更專注於記錄「遇到了什麼困難」以及「最後如何解決」。這是一個結合了 **待辦清單 (To-do List)** 與 **輕量級問題追蹤 (Issue Tracker)** 的整合方案。

---

## ✨ 核心功能
- **視覺化任務管理**：採用靛藍色系與薄荷蘇打配色，支援多級優先度（🔥 緊急、⚡ 普通、🌱 低優先、🏠 家務）。
- **問題追蹤系統**：每個任務可獨立紀錄多筆 Q&A，方便回溯開發痛點。
- **進度即時監控**：滑桿式進度調整，配合動態進度條顯示。
- **雲端雙向同步**：自動同步至 Google Sheets，橫跨桌面與行動裝置。
- **響應式設計**：支援桌面側欄模式與行動裝置緊湊視圖。

---

## 🛠️ 技術架構與創新設計

本專案在雲端同步上採用了 **「GAS Web App 代理模式」**，而非傳統的官方 API 模式。



### 💡 為什麼選擇 Web App 代理模式？
與傳統 API 相比，本專案實現了以下技術突破：
1. **無金鑰化驗證 (Zero API Key)**：
   - 透過 GAS 部署權限設定，前端不需暴露 API Key 即可安全存取。
2. **隱藏敏感資訊 (Hidden IDs)**：
   - 無需在前端公開 Spreadsheet ID。GAS 腳本自動識別所屬試算表，安全性更高。
3. **邏輯解耦 (Logic Decoupling)**：
   - 前端專注於 UI/UX；試算表欄位對位、資料過濾等後端邏輯全數封裝於 GAS 中台。

| 組件 | 技術 |
| :--- | :--- |
| **前端框架** | React 18 (CDN version) |
| **樣式設計** | Tailwind CSS |
| **後端 / 資料庫** | **Google Apps Script (GAS)** |
| **部署環境** | GitHub Pages |
| **本地暫存** | Browser LocalStorage |

## 📈 開發紀錄摘要

- **v1.0 基礎架構**：建立 React 核心邏輯，解決 GAS 跨網域 (CORS) 寫入問題。
- **v2.0 UI 優化**：導入玻璃擬態 (Glassmorphism) 設計與動畫效果。
- **v2.0.5 問題紀錄模組**：實作 Issue Tracker，支援 JSON 格式儲存問答對。
- **v2.0.7 穩定性更新**：
  - 使用 `e.stopPropagation()` 解決刪除按鈕與卡片點擊事件衝突。
  - 修正資料對位錯誤（Sanitization），確保雲端與本地端資料結構一致。

`id` | `text` | `priority` | `progress` | `end` | `description` | `issues` | `isDone`

## 💡 使用說明
1. **新增任務**：點擊右上角「+ 新增任務」，設定名稱與優先級。
2. **追蹤問題**：點擊任務卡片進入詳情，點擊「+ 新紀錄」寫下遇到的問題。
3. **完成任務**：點擊卡片右上角勾選框。
4. **刪除項目**：進入詳情頁面
